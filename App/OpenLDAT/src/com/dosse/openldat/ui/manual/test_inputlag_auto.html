Input Lag measurement

Automated test





This test measures Total System Latency, also known as Input Lag, the time that passes between a button press and something happening on the screen.

To simulate a variety of scenarios that a user might encounter, several settings are available:

  • VSync mode: Vertical synchronization prevents tearing in the image by making sure that the GPU only sends a new frame to the display while the display is not drawing the previous one. This introduces a delay that depends on the framerate, the refresh rate of the screen, and how VSync is implemented.
  • Fake CPU load: This setting simulates a load on the CPU. Games usually take a few milliseconds per frame for all their internal processing before rendering.
  • Fake GPU load: This setting simulates a load on the GPU. This can be used to simulate the complexity of drawing a scene in a game.
  • Test duration: Longer tests will provide more accurate results. 1 minute is recommended in most cases.
Note: Some of these options are only available when using the OpenGL backend. A Swing backend is provided for platforms that don't support OpenGL, but its behavior is controlled by the system.



VSync modes

This test implements the following modes for VSync:

  • Off: This is the mode favored by competitive gamers. It allows the game to run as fast as the hardware allows it to get lower latency, at the cost of image tearing, increased heat, and sometimes bugs.
  • On: Traditional VSync. The application draws to a buffer then waits for the screen to be ready before displaying it. This mode limits the application framerate to the refresh rate of the display, therefore introducing up to a full screen refresh of latency, and the framerate is, that also introduces input delay. Note that the video driver may force triple buffering and/or prerendering of frames, introducing additional delay.
  • On (alternative implementation): Another implementation of VSync that may reduce latency in some cases by letting the desktop compositor handle VSync. May not be supported by all hardware and software combinations.
Note: like most modern games, the test will run in what is usually called "Borderless fullscreen" and not "Exclusive fullscreen".



Fake CPU and GPU loads

Games generally put more load on the system than this test does. These options allows simulating a load on the CPU and GPU during the test, to see how latency changes with it.
Obviously, enabling these options limits the maximum framerate that the application can achieve.



PWM, Strobing and other noise

This test generally gives very accurate results even if PWM backlights are used or some other similar noise is present. However, if possible, it should be turned off for improved accuracy.



Blinking LED

The LED on the front of the sensor device will blink during this test when a click is generated. If you have a high speed camera, you can use this to verify the data reported by this app.
