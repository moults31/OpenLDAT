Input Lag measurement

Manual test

This test allows you to manually test the Total System Latency of a game or even another system, also known as Input Lag.



Setting up the test

To run the test you'll need:

  • A TEST MACHINE running this application, to which the OpenLDAT device is connected
  • A TARGET MACHINE that runs the game or application that you want to measure. This can be any device that can take clicks, but can't run the OpenLDAT application (for instance, a game console, a cloud gaming service, etc.)
  • The MODIFIED MOUSE described in the documentation (optional)
To begin the test:
  • Keep the OpenLDAT device connected to the TEST MACHINE
  • If the TEST MACHINE and the TARGET MACHINE are the same, you don't need the MODIFIED MOUSE and the OpenLDAT device can be used as the click source, otherwise:
    • Connect the 2 wires from the MODIFIED MOUSE to the external input of the OpenLDAT device
    • Connect the USB from the MODIFIED MOUSE to the TARGET MACHINE
  • Position the OpenLDAT device on the screen of the TARGET MACHINE; you need to place it in a place that will change brightness when the mouse is clicked. A muzzle flash is usually a good choice.
During the test:
  • If using the MODIFIED MOUSE, click the mouse at a speed of about 1 click per second
  • Adjust the sensitivity and the green trigger level to make sure the flashes are detected properly
  • Repeat at least 10-20 times to get a good average



General tips
  • For accurate results, get in a situation where clicking the mouse will immediately generate a flash on the screen, without doing any animation first.
  • If you captured some garbage data while adjusting the sensitivity and trigger level, reset the test and repeat.
  • Don't click the mouse too quickly, especially if there's noticeable input lag.
  • Acceptable levels of input lag in video games are below 150ms, anything higher will severly impact your accuracy. The lower the better.
  • If the backlight of the display of the TARGET MACHINE is strobing, it will impact the accuracy of the test. Use the PWM/Strobing detection easily detect this.


PWM, Strobing and other noise

This test generally gives very accurate results even if PWM backlights are used or some other similar noise is present. However, if possible, it should be turned off for improved accuracy.



Blinking LED

The LED on the front of the sensor device will blink during this test when a click is generated. If you have a high speed camera, you can use this to verify the data reported by this app.
