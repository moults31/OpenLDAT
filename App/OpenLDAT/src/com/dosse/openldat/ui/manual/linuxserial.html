Allowing access to serial devices on Linux

To communicate with the OpenLDAT sensor device, your user must be allowed access to serial devices.
Typically, it involves adding your user to one of these groups: uucp, dialout, lock, tty.
Note: your user must be an administrator to do this (must be allowed to run sudo)



Arch, Manjaro, etc.

Open a terminal and run this command:

sudo usermod -a -G uucp $(whoami)

Now reboot the machine and OpenLDAT should work.



Debian, Ubuntu, etc.

Open a terminal and run this command:

sudo usermod -a -G dialout $(whoami)

Now reboot the machine and OpenLDAT should work.



OpenSUSE and similar

Open a terminal and run this command:

sudo usermod -a -G dialout,lock,tty $(whoami)

Now reboot the machine and OpenLDAT should work.



Fedora, Red Hat, etc.

Open a terminal and run this command:

sudo usermod -a -G dialout,lock $(whoami)

Now reboot the machine and OpenLDAT should work.



Other distros

Refer to your distro's documentation on how to access serial devices.
If you can't find it, search for instructions on how to use Arduino IDE on your distro, it will probably explain you how to do it.
If you still can't find anything, look for instructions on how to create custom udev rules, and add a rule to allow all users access to USB devices with VID 4370.

As a last resort, simply run OpenLDAT with sudo.
